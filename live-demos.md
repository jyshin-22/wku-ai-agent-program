# 강의 중 라이브 데모 기획 (강사용)

> 강사가 화면에 직접 시연하는 데모 모음. 각 데모: **목적 · 시점(슬라이드) · 준비물 · 진행 · 예상 결과 · 실패 대비(plan B)**.
> 원칙: 짧게(2~5분), 한 개념당 하나, 실패해도 그 자체가 교육(검증의 중요성). 미리 1회 리허설 권장.

---

## Day 1 — LLM의 원리

### D1-1. 같은 질문, 다른 답 (확률적 생성)
- **목적**: '샘플링/temperature 때문에 매번 답이 다르다'를 눈으로 보여주기
- **시점**: Day1 「샘플링 — 토큰 하나 선택」 / 「같은 모델, 다른 출력」
- **준비물**: 챗봇(ChatGPT/Claude) 창
- **진행**: 동일한 창의적 프롬프트(예: "수면을 주제로 두 줄 시를 써줘")를 3번 연속 실행
- **예상 결과**: 매번 다른 시 → "정답이 아니라 확률"임을 체감
- **plan B**: 미리 캡처해 둔 3개 결과 이미지

### D1-2. 환각 생포하기 (출처를 지어낸다)
- **목적**: 환각이 '그럴듯한 가짜'임을 실시간으로 확인
- **시점**: Day1 「환각(hallucination) 바로 알기」
- **준비물**: 챗봇
- **진행**: 존재하지 않을 법한 요청 — "○○○(가상의 약물)에 대한 2023년 논문 3편을 저자·저널과 함께 알려줘" → 나온 논문을 PubMed/Google에서 같이 검색
- **예상 결과**: 그럴듯하지만 검색하면 없는 논문 → "반드시 교차확인"
- **plan B**: Mata v. Avianca 사건 캡처로 대체 설명
- **주의**: 모델이 "확실치 않다"고 거절할 수도 있음 — 그 경우 "좋은 거절"의 예로 활용

### D1-3. 프롬프트 한 줄의 차이
- **목적**: 프롬프트가 출력을 좌우함을 대비로 보여주기
- **시점**: Day1 「프롬프트가 동작을 바꾼다」 / 「기법 ①~③」
- **준비물**: 챗봇
- **진행**: ① "수면에 대해 설명해줘" → ② "너는 약학 교수야. 약대 2학년에게 수면제의 작용기전을 3가지 핵심으로, 각 1문장으로 설명해줘" 비교
- **예상 결과**: 막연한 답 vs 구조화된 전문 답 — 역할·맥락·형식의 효과
- **plan B**: 캡처 2장 비교

### D1-4. 토큰 쪼개기 보기 (선택)
- **목적**: '토큰'이 글자/단어와 다름을 시각화
- **시점**: Day1 「토큰화」
- **준비물**: OpenAI Tokenizer 웹페이지(platform.openai.com/tokenizer) 또는 유사 도구
- **진행**: 영어 문장과 한글 문장을 각각 넣어 토큰 수 비교
- **예상 결과**: 한글이 더 잘게 쪼개짐 → 비용·길이 함의
- **plan B**: 미리 캡처

## Day 2 — 에이전트 + Claude Code

### D2-1. 에이전트 루프 생중계 (핵심 데모)
- **목적**: 'LLM + 도구 + 반복'이 실제로 도는 모습 보여주기
- **시점**: Day2 「에이전트의 동작 사이클」 / 「AI 에이전트란?」
- **준비물**: Claude Code + `examples/mini-project/sandbox/`
- **진행**: sandbox에서 `claude` 실행 → "expenses.csv를 항목별로 합산하는 파이썬 스크립트를 만들고 실행해서 결과를 보여줘" → 에이전트가 파일 읽기→코드 작성→실행→결과 출력하는 과정을 그대로 노출
- **예상 결과**: 관찰→행동→관찰 루프가 화면에 그대로 (식비 44,500 등 합계)
- **plan B**: 미리 녹화한 터미널 캡처/영상

### D2-2. 바이브코딩 — 말로 고치기
- **목적**: 자연어로 반복 수정하는 흐름 체감
- **시점**: Day2 「바이브코딩 사이클 ①~④」
- **준비물**: Claude Code (D2-1에 이어서)
- **진행**: "표를 금액 내림차순으로 정렬하고, 총합도 마지막 줄에 추가해줘" → 결과 확인 → "총합 글자를 굵게" 식으로 1~2회 더
- **예상 결과**: 작은 단위 반복 + 매번 실행 확인 = 검증 습관
- **plan B**: 캡처

### D2-3. 도구 사용 — 검색하는 AI (선택)
- **목적**: function calling/도구 호출 개념을 실제로
- **시점**: Day2 「도구(tools)」
- **준비물**: 웹검색 가능한 챗봇(ChatGPT Search 등)
- **진행**: "오늘 기준 ○○ 약물의 최신 FDA 상태를 출처와 함께 알려줘" → 검색 호출·출처 인용 노출
- **예상 결과**: 모델이 스스로 검색 도구를 호출 → 출처 링크
- **plan B**: 캡처

### D2-4. 설치 점검 라이브 (진행성)
- **목적**: 설치가 끝났는지 다 같이 확인
- **시점**: Day2 오후 설치/설정 실습
- **준비물**: 각자 노트북
- **진행**: 강사가 화면에 `claude --version` → `/help` → sandbox 동작 확인을 시연하고 학생이 따라 함
- **plan B**: 트러블슈팅 슬라이드(OS별)

## Day 3 — 미니 프로젝트 + 리서치 이론

### D3-1. 미니 프로젝트 0→1 (기획서 → 동작)
- **목적**: 기획서 한 장에서 실제 동작물까지 한 호흡에
- **시점**: Day3 오전/오후1 (개인 프로젝트 시작 전 시범)
- **준비물**: Claude Code + `examples/mini-project/project-plan-template.md`
- **진행**: 채운 기획서를 주고 "이 기획서대로 만들어줘" → 동작 확인 → 작은 개선 1회
- **예상 결과**: 학생들이 자기 프로젝트를 시작하는 틀을 봄
- **plan B**: sandbox 데모(D2-1)로 대체

### D3-2. 생성 vs 선별 미니 토론 시연 (선택)
- **목적**: 가설을 '많이 만들기'를 즉석에서 보여주기
- **시점**: Day3 오후2 「생성 vs 선별」 쟁점
- **준비물**: 챗봇
- **진행**: "수면-인지 분야에서 검증 가능한 새 가설 후보를 8개 빠르게 만들어줘" → 다양성 확인 → "이 중 검증 비용이 낮고 새로운 것 2개만 골라줘"
- **예상 결과**: breadth(생성) → selection(선별)의 실제 모습
- **plan B**: SAMPLE 제안서로 설명

## Day 4 — 가설 생성 에이전트 (핵심 데모 데이)

### D4-1. 논문 수집 라이브 (PubMed)
- **목적**: 진짜 논문이 폴더로 들어오는 과정 + '실제 논문 확인'
- **시점**: Day4 오전 「PubMed에서 논문 모으기」 / 수집 실습 직전
- **준비물**: Claude Code + `examples/research-agent/collect/collect_pubmed.py` (인터넷)
- **진행**: `python3 collect_pubmed.py "metformin AND aging" 10` 실행 → `papers/` 생성 → 한 편을 PMID 링크로 열어 실재 확인
- **예상 결과**: 초록·PMID가 담긴 파일들 + 링크가 진짜로 열림
- **plan B**: 미리 받아둔 `papers/` 폴더 (네트워크/PubMed 장애 대비) — **반드시 미리 준비**
- **주의**: 강의 당일 PubMed 응답 지연 가능 → 개수 5~10으로 작게

### D4-2. gap 발견 → 가설 생성 라이브 (하이라이트)
- **목적**: corpus의 모순에서 새 가설이 나오는 핵심 흐름
- **시점**: Day4 오후 「자료 파악/gap」 「가설 생성」
- **준비물**: Claude Code + `examples/research-agent/`(CLAUDE.md + sources/)
- **진행**: research-agent 폴더에서 `claude` → "sources/를 같은 틀로 정리하고, 주장이 충돌하는 gap을 찾아줘" → 이어서 "그 gap에서 검증 가능한 새 가설을 6개 만들고 각 근거를 달아줘"
- **예상 결과**: doc01 vs doc02 충돌 → 단시간 수면자 가설 등 (SAMPLE과 유사)
- **plan B**: `output/SAMPLE-hypothesis-proposal.md`를 띄워 설명

### D4-3. 가설 평가·우선순위 (선택, D4-2에 이어서)
- **목적**: 생성한 가설을 기준으로 추리는 모습
- **진행**: "위 가설들을 근거·새로움·타당성·검증가능성으로 평가해 순위를 매기고 상위 2개를 추천해줘"
- **예상 결과**: breadth→selection 마무리
- **plan B**: SAMPLE의 4장 참고

## Day 5 — 제안서·발표

### D5-1. AI로 제안서 초안 만들기
- **목적**: 분석 노트 → 제안서 초안을 AI가 빠르게
- **시점**: Day5 오후1 「AI로 자료 만드는 흐름」
- **준비물**: Claude Code + `output/report-template.md` + (예시로) SAMPLE 분석 노트
- **진행**: "이 분석 노트와 양식을 바탕으로 가설 제안서 초안을 만들어줘" → 생성 → **사실·출처를 같이 점검**(검증 강조)
- **예상 결과**: 빠른 초안 + "초안 ≠ 최종본, 사람이 검증"
- **plan B**: SAMPLE 제안서를 결과물로 제시
