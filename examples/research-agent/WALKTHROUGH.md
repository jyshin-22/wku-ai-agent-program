# 리허설(mock) — 리서치 에이전트 처음부터 끝까지 한 번 해보기

> **목적**: 진짜 팀 프로젝트를 시작하기 전에, **제공된 가상 자료(`sources/`)**로
> 자료파악 → gap발견 → 가설생성 → 평가 → 제안서까지 **전 과정을 한 번 완주**해 본다.
> 인터넷·논문 수집 없이(오프라인) 가능하며, 이 폴더 하나만 있으면 된다.

## 이번 리허설의 설정 (그대로 따라 하기)
- **분야**: 수면과 인지·생산성
- **발견 목표**: `sources/` 자료의 모순·빈틈에서 **검증할 만한 새 가설**을 찾아 제안
- **자료(corpus)**: 이미 들어 있는 `sources/doc01~doc06` (가상 자료 6편, 일부는 일부러 상충·저신뢰로 설계됨)

## 끝나면 손에 남는 것 (산출물)
1. `research-plan.md` — 채운 기획서
2. `worksheets/1-literature-matrix.md` — 논문 정리표 (채움)
3. `worksheets/2-gap-list.md` — gap 목록 (채움)
4. `worksheets/3-hypothesis-eval.md` — 가설 후보·평가표 (채움)
5. `output/report.md` — **rationale 갖춘 새 가설 제안서** (최종 결과물)

## 단계 1 — 기획서 채우기 (5분)
**무엇/왜**: 무엇을 발견하려는지 먼저 정해야, 에이전트가 방향을 잃지 않는다.

**학생이 칠 프롬프트**:
```
research-plan-template.md 를 복사해서 research-plan.md 를 만들고 아래 값으로 채워줘.
- 분야: 수면과 인지·생산성
- 발견 목표: sources 자료의 모순·빈틈에서 검증 가능한 새 가설을 찾아 제안
- corpus: sources/ 폴더 (가상 자료 6편)
- 스코프: 자료파악 → gap발견 → 가설생성 → 평가·제안 (실제 실험 검증은 제외)
```
✅ **확인**: `research-plan.md` 파일이 생기고 `[ ]` 자리가 위 값으로 채워졌는지 연다.
📄 **산출물**: `research-plan.md`

## 단계 2 — 자료 파악 (논문 정리표) (10분)
**무엇/왜**: 자료를 **같은 틀**로 정리해야 다음 단계에서 모순·빈틈이 눈에 보인다.

**학생이 칠 프롬프트**:
```
sources/ 폴더의 doc01~doc06을 모두 읽고, worksheets/1-literature-matrix.md 의 표 형식
(제목·연도·핵심 주장·방법·데이터·근거 강도·가정/한계·출처)으로 한 논문당 한 행씩 정리해줘.
근거가 약하거나 출처 신뢰도가 낮은 자료(예: 블로그)는 '근거 강도'에 그렇게 표기해줘.
정리한 표로 worksheets/1-literature-matrix.md 를 업데이트해줘.
```
✅ **확인 포인트**:
- 6개 문서가 모두 한 행씩 들어갔는가
- **doc06(블로그)**의 근거 강도가 '약'으로 표시됐는가
- doc01(7–9시간)과 doc02(6시간)의 주장이 **다르게** 적혔는가 (→ 다음 단계 gap 후보)

📄 **산출물**: `worksheets/1-literature-matrix.md`

## 단계 3 — gap 발견 (10분)
**무엇/왜**: gap(모순·미검증·빠진 관점)이 **새 가설의 원천**이다.

**학생이 칠 프롬프트**:
```
방금 만든 논문 정리표를 근거로, 자료들 사이의 gap을 찾아줘.
- 모순(서로 다른 주장), 미검증(근거 약함), 빠진 관점(안 다뤄진 조건·집단)으로 분류하고
- 각 gap에 반드시 근거 논문(doc 번호)을 달아줘.
- 신뢰도 낮은 블로그(doc06)는 gap의 근거로 쓰지 말고, 오히려 '검증 대상'으로 다뤄줘.
결과로 worksheets/2-gap-list.md 를 채워줘.
```
✅ **확인 포인트**:
- **doc01 vs doc02의 수면시간 모순**(7–9h vs 6h)이 잡혔는가
- 낮잠(doc05)이나 카페인→생산성(doc03·doc04) 같은 **빠진 관점**이 하나라도 나왔는가
- 각 gap에 근거 doc 번호가 붙어 있는가 (근거 없는 gap은 실패)

📄 **산출물**: `worksheets/2-gap-list.md`

---

## 단계 4 — 가설 생성 (다수) (10분)
**무엇/왜**: gap에서 **여러 개**를 만든다. 하나에 집착하지 않고 후보군을 확보하는 게 핵심.

**학생이 칠 프롬프트**:
```
gap 목록을 바탕으로 검증 가능한 새 가설을 5개 만들어줘. 규칙:
- 각 가설은 '~한 조건에서 ~할 것이다' 한 문장으로.
- 두 방식을 섞어서: (a) gap 기반(모순·미검증·빠진 관점), (b) 조합·전이(다른 자료의 아이디어 결합).
- 각 가설에 [근거: doc번호/어떤 gap] 을 반드시 달아줘.
- 이미 잘 알려진 사실은 제외하고, 근거 없는 막연한 상상도 제외해줘.
```
✅ **확인 포인트**:
- 가설이 **5개 정도** 나왔는가
- 각 가설이 **한 문장 · 검증 가능** 형태인가
- 각 가설에 **근거(doc·gap) 태그**가 붙어 있는가

📄 (다음 단계에서 평가표로 옮긴다)

## 단계 5 — 평가·우선순위 (10분)
**무엇/왜**: 많이 만들었으니, **무엇을 먼저 검증할지** 정해야 한다.

**학생이 칠 프롬프트**:
```
방금 만든 가설들을 worksheets/3-hypothesis-eval.md 의 표에 옮기고,
영향(impact)·타당성(plausibility)·테스트 비용·새로움을 상/중/하로 평가한 뒤,
'영향 × 타당성 ÷ 비용'을 기준으로 순위를 매겨줘.
각 가설에 한 줄짜리 '검증 아이디어'도 붙여줘. 상위 1~2개를 제안 후보로 표시해줘.
결과로 worksheets/3-hypothesis-eval.md 를 채워줘.
```
✅ **확인 포인트**:
- 모든 가설에 4개 기준 점수가 매겨졌는가
- **상위 1~2개**가 제안 후보로 표시됐는가
- 각 가설에 검증 아이디어가 한 줄이라도 있는가
- (사람의 판단) 에이전트가 매긴 순위가 납득되는가 — **그대로 믿지 말고 한 번 검토**

📄 **산출물**: `worksheets/3-hypothesis-eval.md`

## 단계 6 — 제안서 작성 (최종 결과물) (10분)
**무엇/왜**: 상위 가설을 **rationale과 함께** 하나의 문서로 정리한다. 이게 진짜 결과물.

**학생이 칠 프롬프트**:
```
output/report-template.md 형식으로 output/report.md 를 만들어줘.
- 발견한 gap을 근거(출처)와 함께 정리하고
- 상위 가설 1~2개를 우선순위 순으로: 근거(gap·출처) / 새로움·중요성 / 검증 아이디어 / 우선순위 이유
- 마지막에 자료의 한계(가상 자료·표본 등)와 향후 검증에 필요한 것을 적어줘.
```
✅ **확인 포인트**:
- 최상위 가설이 **한 줄 '핵심 제안'**으로 요약됐는가
- 각 가설의 **근거가 doc·gap으로 추적 가능**한가
- '한계' 섹션에 **가상 자료라는 점**이 적혔는가

📄 **산출물**: `output/report.md` ← **이게 리허설의 완성품**

> 비교용: 이미 만들어진 완성 예시가 `output/SAMPLE-hypothesis-proposal.md` 에 있다.
> 내 결과와 비교해보면 좋다 (똑같을 필요는 없다).

- **에이전트가 근거 없는 가설을 지어냄** → "각 가설에 sources의 doc 번호 근거를 반드시 붙여줘. 근거 없으면 빼줘."
- **한 가설만 고집** → "후보를 최소 4개 유지하고, 하나로 좁히지 마."
