# 리서치 에이전트용 프롬프트 템플릿

리서치 흐름 **질문 → 검색 → 검증 → 종합 → 보고** 의 단계별 프롬프트.
복사해서 `[ ]` 를 채워 쓰세요.

## 1. 질문 분해
```
연구 질문: "[핵심 질문]"
이 질문을 답하려면 어떤 하위 질문들이 필요한지 3~6개로 분해해줘.
각 하위 질문은 검색 가능한 형태로, 서로 겹치지 않게.
```

## 2. 검색·수집
```
다음 하위 질문에 답할 자료를 sources/ 폴더(또는 웹)에서 찾아줘.
- 하위 질문: "[질문]"
각 자료마다 (제목 · 출처 · 날짜 · 핵심 주장 1~2개)를 표로 정리하고,
원문 위치(파일명/URL)를 반드시 함께 적어줘.
```

## 3. 검증·교차확인
```
아래 주장들에 대해 검증해줘.
- 주장: "[주장]"
1) 이 주장을 뒷받침하는 자료와 반박하는 자료를 각각 찾아줘.
2) 출처의 신뢰도(학술/기관 vs 개인 블로그 등)를 표시해줘.
3) 자료 간 상충이 있으면 명확히 짚어줘.
근거가 약하거나 출처가 불분명하면 "불확실"로 표시.
```

## 4. 종합
```
지금까지 수집·검증한 내용을 종합해 "[핵심 질문]"에 답해줘.
- 결론은 근거(출처)와 함께
- 자료 간 의견이 갈리는 부분은 양쪽을 모두 제시
- 확실하지 않은 부분은 "한계/불확실"로 따로 표기
```

## 5. 보고
```
종합 결과를 보고서 형식으로 정리해줘.
구조: 개요 → 핵심 질문 → 발견(근거·출처 포함) → 한계 → 결론
각 주장 옆에 [출처: 파일명/URL] 표기. 1~2장 분량.
```

> 팁: 한 번에 다 시키지 말고 단계별로. 각 단계 결과를 사람이 검토한 뒤 다음 단계로.
