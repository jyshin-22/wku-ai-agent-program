# 리서치 에이전트용 프롬프트 템플릿 (가설 생성)

흐름 **자료 파악 → gap 발견 → 가설 생성 → 평가·우선순위 → 제안** 의 단계별 프롬프트.
복사해서 `[ ]` 를 채워 쓰세요. (한 번에 다 시키지 말고 단계별로, 사람이 검토하며)

## 1. 자료 파악
```
관심 분야 / 발견 목표: "[분야]"
sources/ 폴더의 자료를 읽고, 각 문서의 (핵심 주장 · 근거 · 출처 · 신뢰도)를 표로 정리해줘.
이미 합의된 것과 아직 불확실한 것을 구분해줘.
```

## 2. gap·모순 발견
```
정리한 자료에서 '빈틈(gap)'을 찾아줘:
- 자료 간 서로 충돌하는 주장
- 근거가 약하거나 검증되지 않은 주장
- 다뤄지지 않은 관점·조건
각 gap마다 어느 자료에서 나왔는지(근거)를 표기해줘.
```

## 3. 가설 생성 (다수)
```
위 gap·모순을 바탕으로 검증할 만한 **새 가설을 6~10개** 만들어줘.
- 각 가설은 한 문장, 검증 가능한 형태로
- 어떤 gap에서 나왔는지 근거를 달고
- 우리가 가진 데이터/도구로 확인 가능한 범위로 제약
- 이미 잘 알려진 것은 제외
```

## 4. 가설 평가·우선순위
```
생성한 가설들을 다음 기준으로 평가해 점수·순위를 매겨줘:
- 근거(gap·출처) / 새로움 / 중요성 / 타당성·검증 가능성
표로 정리하고 상위 1~2개를 추천해줘. 왜 그렇게 매겼는지 rationale도.
```

## 5. 가설 제안서
```
우선순위 상위 가설을 제안서로 정리해줘.
구조: 분야·발견 목표 → 발견한 gap(근거) → 제안 가설(가설·근거·새로움·검증 아이디어·우선순위) → 한계.
각 항목에 [근거: 파일명] 표기.
```

> 팁: 3단계에서 '많이·다양하게', 4단계에서 '추리기' — 강의에서 본 '생성 vs 선별'을 직접 적용해 보세요.
