## 기존 연구 프로세스
1. 분야 전체에 대해 공부한다
2. 그 안에서 세부 관심 분야를 설정한다
3. 세부 관심 분야의 논문들을 선별한다 (전체 논문 및 우수 논문)
4. 세부 관심 분야의 논문들을 검증 및 재현해본다
5. 세부 관심 분야의 논문들을 개선할 가설을 수립한다
	* 완전히 새로운 가설을 수립한다
		* 배경 지식을 바탕으로
		* 기존 논문을 바탕으로
	* 기존 연구의 단점을 보완하는 가설을 수립한다
		* 배경 지식을 바탕으로
		* 기존 논문을 바탕으로
6. 수립된 가설에 우선순위를 매기고 높은 우선순위부터 검증한다
	* 배경 지식을 바탕으로
	* 기존 논문을 바탕으로

## Agentic 프로세스 질문들
1. 전체 분야에 대한 공부 어떻게 하는건지?
2. 관심분야의 논문들 중에서 좋은 논문들을 어떻게 선별하는지? (선별해야하긴 하나?)
3. 관심분야의 논문들에 대한 공부를 어떻게 하는건지?
4. 논문들을 개선할 가설을 어떻게 도출할것인지?
5. 생성된 가설들을 어떻게 우선순위 할건지?

Agent 기반 가설 생성의 쟁점 - 둘 중 어느 것에 집중해야 하는가?
* 성과가 좋을 것으로 기대되는 새로운 가설을 높은 비용의 테스트를 거치지 않고도 어떻게 선별할 것인가
* 또는, 성과가 좋은 새로운 가설을 만들 확률 자체를 높이기 위해 최대한 많은 가설을 만들게 할 것인가?
* 생각해볼 만한 포인트
	* 에이전트를 활용하면 테스트 자체의 비용도 획기적으로 줄어든다
	* 만들어지는 가설 자체를 우리가 쓸 수 있는 데이터/도구 범주로 제한하고 가설을 최대한 많고 다양하게 만드는 것이 좋을까?

## 답변 및 권고 (Claude)

### 기존 연구 프로세스 — 평가
골격은 맞음. 단 agent 자동화를 위해 4가지 보정:
1. **선형 → 반복 루프**: 가설·검증 결과가 "어떤 문헌이 중요한가"를 바꾼다.
2. **'재현' → '검증'**: 전체 재현은 대부분 비용상 불가 → 주장–근거 점검·가정/한계 식별로 재정의.
3. **'질문 정의'와 'gap/모순 분석' 명시**: 연구의 단위는 분야가 아니라 질문. 가설의 최대 원천은 문헌 간 모순·공백.
4. **feasibility(가용 데이터·도구로 테스트 가능?)를 가설 생성 단계의 1급 제약으로**.

### Agentic 질문 1~5 — 요지
1. **분야 공부**: 전부 읽지 말고 *맵 생성* — review/landmark에서 출발, LLM 지식을 scaffold로 + RAG로 교정, taxonomy·쟁점·concept graph 산출, saturation에서 정지.
2. **논문 선별(필요한가?)**: 필요하되 funnel로 — 넓게 수집→abstract 스크린→랭킹→상위+**모순/outlier**만 정독. 기준은 *질문 적합도+다양성*, citation은 한 feature일 뿐(후행·편향 지표).
3. **논문 학습**: '읽기'가 아니라 *동일 schema로 구조화 추출* + claim/모순 그래프. 관계 구조에서 gap이 드러난다.
4. **가설 도출**: 구조적 generator — gap/모순·조합·전이·가정 완화·scaling·failure-mode. 각 가설에 출처 태그(추적·검증 가능).
5. **우선순위**: impact × plausibility ÷ 비용, feasibility로 게이트. 단 LLM 랭킹은 노이즈·편향 → ensemble·싼 경험적 사전스크린·portfolio로 보정.

### 핵심 쟁점 — '다수 생성' vs '선별'
**테스트가 싸지는 조건에선 '제약된 다수 생성 + 싼 경험적 필터'(breadth)로 기울일 것.**
- **이유**: a priori로 winner를 맞히는 건 LLM이 가장 약한 부분(환각·miscalibration). 싼 테스트가 가능하면 *노이즈 generator + 현실 grounded 필터*가 '똑똑한 selector'를 이긴다(bitter-lesson류).
- **'가용 데이터/도구로 제약'(본인 직관)이 핵심 enabler**: 제약이 ① 싼 테스트를 실제 가능케 하고 ② 탐색공간을 묶는다. 제약 없는 다양성 극대화는 테스트 불가 가설을 양산 → 싼 필터 소멸 → 비싼 a priori 선별로 회귀.
- **가드 3가지**: 다중검정 위양성(hold-out·재현·통계 보정) · 싼 테스트는 proxy(Goodhart 주의) · 대규모 생성 비용.
- **funnel**: 넓게 생성(제약) → LLM 최소 prune(winner 고르기 금지) → 싼 경험적 스크린(실제 선별) → 소수만 비싼 검증.
- ⚠ **도메인 caveat**: 신약/wet-lab/임상은 *최종* 검증이 agent로도 안 싸진다. agent가 싸게 하는 건 in-silico/문헌/시뮬 단계 → 전략은 **단계적**(싼 in-silico breadth → 좁히기 → 비싼 실측).

- **MOOSE** (ACL 2024) — open-domain 가설 발견(화학 비한정). 약학/일반 주제에 더 잘 맞는 선행 연구. github.com/ZonglinY/MOOSE
